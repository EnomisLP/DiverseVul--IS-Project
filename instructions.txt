0. WHAT THIS PROJECT IS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

A binary classifier for C/C++ functions: vulnerable (1) vs non-vulnerable (0).

Two parallel tracks:

  Track A — classical baseline
    EXP_0  Logistic Regression on TF-IDF features
    EXP_1  Random Forest on TF-IDF features

  Track B — deep learning (HEFT: two-phase PEFT)
    EXP_2  Frozen NeoBERT backbone + linear head (probe)
    EXP_3  NeoBERT + LoRA on W_q and W_v, rank ∈ {8, 16}
    EXP_4  Freeze EXP_3 LoRA → apply ReFT on layers {12,16,20,24}, rank ∈ {4, 8}
    EXP_5  Identical to EXP_4 with ModernBERT-Base backbone

Dataset: RDiverseVul (Refined DiverseVul, Feb 2025).
Methodology: KDD. Evaluation: Stratified 5-fold CV + locked 20% holdout.
Primary metric: PR-AUC (dataset is class-imbalanced; accuracy is not used).

      My Drive/vuln-detection-data/

  In the first code cell of 00_setup.ipynb (or manually):

      !git clone https://github.com/<org>/<repo>.git /content/vuln-detection

  Still in 00_setup.ipynb:

  See section 4 below.

  Every time Colab disconnects or the runtime restarts, you must redo

4. RUNNING THE EXPERIMENTS — NOTEBOOK ORDER
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Run notebooks strictly in numerical order. Each one depends on outputs
produced by the previous ones.

────────────────────────────────────────────────────
00_setup.ipynb  — Environment bootstrap
────────────────────────────────────────────────────
What it does:
  - Clones the repo and installs dependencies (if not already done)
  - Verifies GPU availability
  - Downloads/checks the NeoBERT tokenizer and model weights from HuggingFace
    (this caches them in ~/.cache/huggingface and avoids re-downloading later)
  - Prints a system info summary (CUDA version, RAM, disk)

When to re-run: Only at the start of a new Colab session, or after
adding a new dependency to requirements.txt.

────────────────────────────────────────────────────
01_track_a.ipynb  — Track A: TF-IDF + LR + RF
────────────────────────────────────────────────────
What it does:
  1. Loads RDiverseVul via src.data.load_rdiversevul
  2. Creates the holdout split via src.data.make_holdout_split
     (saves train_idx.npy and holdout_idx.npy to data/splits/)
     IMPORTANT: this must happen exactly once. If you re-run this cell
     a second time in the same session it will raise FileExistsError —
     this is intentional. Pass overwrite=True only if you want a new split.
  3. Runs 5-fold CV for EXP_0 (Logistic Regression) and EXP_1 (Random Forest)
  4. Writes per-fold JSON files to results/metrics/
  5. Saves PR curve PNGs to results/figures/

Outputs committed to Git:
  results/metrics/EXP_0_fold*.json
  results/metrics/EXP_0_aggregate.json
  results/metrics/EXP_1_fold*.json
  results/metrics/EXP_1_aggregate.json
  results/figures/EXP_0_pr_curves.png
  results/figures/EXP_1_pr_curves.png

Do NOT commit: data/splits/ (gitignored)

────────────────────────────────────────────────────
02_track_b_probe.ipynb  — EXP_2: frozen NeoBERT probe
────────────────────────────────────────────────────
What it does:
  1. Loads the split indices created in 01 (not the raw data again)
  2. Loads NeoBERT with both bug fixes (use_unpadding=False, float32 head)
  3. Freezes the backbone; trains only the 2-class linear head
  4. 5-fold CV → per-fold JSON metrics

Purpose: establishes a strong baseline for what NeoBERT "already knows"
without any fine-tuning. If EXP_3 does not improve on EXP_2, LoRA is not
adding value.

────────────────────────────────────────────────────
03_track_b_lora.ipynb  — EXP_3: LoRA fine-tuning
────────────────────────────────────────────────────
What it does:
  1. For each rank r ∈ {8, 16} (from configs/heft.yaml):
     a. Injects LoRA adapters into W_q and W_v of NeoBERT
     b. Trains for 3 epochs with AdamW + linear warmup/decay
     c. Saves adapter weights (NOT the full model) to:
            results/checkpoints/EXP_3_r{r}/fold{N}/
     d. Writes per-fold JSON metrics
  2. Selects the best rank by mean CV PR-AUC
  3. Prints the path to the best fold checkpoint — you will need this for 04

After this notebook: pick the best EXP_3 checkpoint directory and note it
down. It is passed into 04_track_b_heft.ipynb as lora_checkpoint_dir.

────────────────────────────────────────────────────
04_track_b_heft.ipynb  — EXP_4 + EXP_5: full HEFT
────────────────────────────────────────────────────
What it does:
  1. For each (lora_r, reft_r) combination from configs/heft.yaml:
     a. Loads the frozen EXP_3 LoRA checkpoint (DO NOT retrain LoRA here)
     b. Applies LoreftIntervention on hidden states at layers {12,16,20,24}
     c. Trains ReFT parameters only (LoRA is fully frozen)
     d. Saves ReFT adapter weights to:
            results/checkpoints/EXP_4_lorar{lora_r}_reftr{reft_r}/fold{N}/
  2. Repeats the above with backbone swapped to ModernBERT-Base (EXP_5)
  3. Writes all metrics to results/metrics/

You MUST set lora_checkpoint_dir at the top of this notebook to the best
EXP_3 checkpoint path identified in 03_track_b_lora.ipynb.

────────────────────────────────────────────────────
05_results.ipynb  — Final holdout evaluation + comparison
────────────────────────────────────────────────────
What it does:
  1. Loads all CV aggregate metrics from results/metrics/
  2. Selects the best hyperparameters per experiment based on CV PR-AUC
  3. Calls evaluate_on_holdout for each experiment (touches the locked 20%
     for the FIRST and ONLY time)
  4. Produces comparison tables (Track A vs Track B)
  5. Saves holdout PR curves to results/figures/

CRITICAL: Do not run this notebook — or any holdout evaluation — until all
CV hyperparameter decisions are finalised. The holdout is a one-shot test.
Running it to "check progress" invalidates the experimental protocol.

7. CONFIG FILES AND WHAT THEY CONTROL
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

configs/track_a.yaml
  Controls all Track A behaviour. Sections:
    tfidf              — vocabulary size, n-gram range, sublinear TF
    logistic_regression — penalty, C, solver (EXP_0)
    random_forest      — n_estimators, max_features, criterion (EXP_1)
    cv                 — n_splits, test_size (holdout fraction), random_state

configs/track_b.yaml
  Controls Track B model loading and training. Sections:
    model    — HuggingFace model name (swap here for EXP_5), max_seq_length,
               use_unpadding (must be false), attention_output_dtype (float32)
    training — epochs, batch_size, lr, weight_decay, warmup_ratio, fp16
    cv       — same structure as track_a.yaml cv section

configs/heft.yaml
  Controls LoRA and ReFT hyperparameters. Sections:
    lora   — target_modules, r (list of ranks), lora_alpha, lora_dropout, bias
    reft   — layers, r (list of ranks), prefix_length, intervention_type,
             position, lr, epochs

  NOTE: lora.r and reft.r are LISTS. The training loop iterates over them
  automatically. To run a single rank, change the list to a single value,
  e.g., r: [8] instead of r: [8, 16].

8. THE SPLIT SYSTEM — CRITICAL RULES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

The data is split exactly once into:
  80% training set  → used for CV (5 folds) and final model retraining
  20% holdout set   → touched only in 05_results.ipynb

This split is produced by make_holdout_split() in 01_track_a.ipynb and saved
as numpy index arrays in data/splits/:
  train_idx.npy     — integer row indices for the 80% training partition
  holdout_idx.npy   — integer row indices for the 20% holdout partition
  split_meta.json   — split parameters and label balance statistics

These files are GITIGNORED. They are regenerated from the raw dataset at the
start of each session by running cell 2 of 01_track_a.ipynb. If the files
already exist (i.e., you ran the notebook twice), make_holdout_split() will
raise FileExistsError — this prevents accidental re-splitting which would
invalidate comparisons between experiments.

The random_state in the config (default 42) guarantees the same split is
produced every time from the same dataset. Do not change it mid-project.

10. COMMON ISSUES AND FIXES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

"ModuleNotFoundError: No module named 'src'"
  → You forgot to run `pip install -e .` after cloning.
    Run it and restart the kernel.

"FileExistsError: Split files exist"
  → make_holdout_split() already ran this session.
    Call load_holdout_split() instead, or pass overwrite=True
    only if you genuinely want a new random split.

"FileNotFoundError: No .csv or .jsonl in data/raw/"
  → The dataset file was not copied into data/raw/.
    Follow section 3, Step 6.

"CUDA out of memory"
  → Reduce batch_size in configs/track_b.yaml (try 4 or 2).
    Alternatively, reduce max_seq_length (try 512 or 1024 for debugging,
    but use 4096 for final runs to match the paper).

NaN loss during Track B training
  → Both bug fixes in model.py must be active:
      use_unpadding: false         (configs/track_b.yaml)
      attention_output_dtype: float32   (configs/track_b.yaml)
    If you are loading a checkpoint, verify it was saved with these fixes.

"RuntimeError: ReFT has not been applied to this model"
  → You called reft_forward() or save_reft_adapter() without first calling
    apply_reft(). Check the setup cell in 04_track_b_heft.ipynb.

"RuntimeError: Found N trainable LoRA parameter(s)"
  → apply_reft() detected that freeze_lora() was not called before Phase 2.
    Call lora.freeze_lora(model) before lora.apply_reft(model, ...).

Colab session disconnected mid-training
  → Results up to the last completed fold are already saved. You can resume
    by skipping folds that already have checkpoint directories.
    Checkpoints in Google Drive are safe. Rerun only the interrupted notebook.

NeoBERT weights downloading slowly / failing
  → Run 00_setup.ipynb once on a stable connection to cache the weights in
    ~/.cache/huggingface. Subsequent runs will load from cache.
